# Product Requirements Register

## Purpose

Create a minimum traceable requirements register for `xi-io Inbox` so future wargame, UI, architecture, and runtime slices can link work to product intent and validation evidence.

## Requirement Status Values

| Status | Meaning |
| --- | --- |
| planned | documented but not implemented |
| partial | partly represented in docs or static preview |
| blocked | cannot proceed until named blocker clears |
| deferred | intentionally later scope |
| satisfied-docs | documented enough for current planning pass |
